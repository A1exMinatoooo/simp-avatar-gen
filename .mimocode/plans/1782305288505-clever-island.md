## Feature 2: 修复中文字体加载问题

**根因**: `AvatarPreview.tsx:33` 在 `requestAnimationFrame` 中调用 `renderAvatar()`，但没有等待字体加载完成。切换 Google Font 时，浏览器异步下载字体文件，canvas 在字体就绪前就绘制了，导致中文回退到系统字体。

**修复**: 在 `AvatarPreview.tsx` 的 `performRender` 中，调用 `renderAvatar` 前 await `document.fonts.ready`：

```ts
renderTimerRef.current = requestAnimationFrame(async () => {
  setIsRendering(true)
  await document.fonts.ready  // 等待字体加载完成
  renderAvatar(...)
  setIsRendering(false)
})
```

`document.fonts.ready` 在字体已加载时立即 resolve，零开销。

**修改文件**: `src/components/AvatarPreview.tsx`

## Feature 3: 图片上传 + 文字叠加

### 新增类型 (`src/types.ts`)

```ts
type BackgroundMode = 'color' | 'image'
interface TextPosition { x: number; y: number }  // 0-1 归一化坐标
interface TextOverlay { bgColor: string; bgOpacity: number }  // 0-1
```

扩展 `AvatarConfig`: 新增 `bgMode`, `imageDataUrl`, `textPosition`, `textOverlay` 字段。

### Canvas 渲染 (`src/utils/canvas.ts`)

`renderAvatar` 新增可选 `options` 参数：
- `bgMode === 'image'` 时：居中裁剪绘制图片到 500x500
- 绘制文字背景半透明矩形（颜色+透明度可配）
- 文字位置由 `textPosition`（归一化坐标）决定

### 新增组件

| 组件 | 文件 | 功能 |
|------|------|------|
| `BackgroundModeToggle` | `src/components/BackgroundModeToggle.tsx` | 纯色/图片模式切换 |
| `ImageUploader` | `src/components/ImageUploader.tsx` | 文件上传、裁剪为 500x500 data URL、预览/删除 |
| `OverlaySettings` | `src/components/OverlaySettings.tsx` | 叠加背景颜色选择 + 透明度滑块 |
| `DraggableText` | `src/components/DraggableText.tsx` | canvas 上方透明交互层，拖拽定位文字 |

### 拖拽实现 (`DraggableText`)

- 在 canvas 上方覆盖透明 div，使用 Pointer Events API
- `pointerdown` 捕获指针，`pointermove` 计算归一化坐标（clamp 0.1-0.9）
- `setPointerCapture` 确保拖拽可靠，支持触屏
- 仅在图片模式下启用
